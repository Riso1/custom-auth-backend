# Система аутентификации и авторизации пользователей

## Описание проекта

Проект представляет собой backend-приложение на Django REST Framework для регистрации пользователей, аутентификации по JWT-токенам и разграничения доступа на основе ролей.

В системе реализованы:

- регистрация пользователей;
- авторизация по email и паролю;
- выход из системы;
- просмотр и редактирование профиля;
- мягкое удаление пользователей;
- ролевая модель доступа;
- управление правилами доступа через API;
- проверка прав на выполнение действий над ресурсами.

## Используемые технологии

- Python
- Django
- Django REST Framework
- SQLite
- PyJWT

## Установка и запуск

Создать виртуальное окружение:

```bash
python -m venv venv
```

Активировать окружение:

Windows:

```bash
venv\Scripts\activate
```

Установить зависимости:

```bash
pip install -r requirements.txt
```

Применить миграции:

```bash
python manage.py migrate
```

Запустить сервер:

```bash
python manage.py runserver
```

## Структура системы доступа

### Роли

В системе предусмотрены следующие роли:

- Пользователь
- Менеджер
- Администратор

### Ресурсы

- Пользователи
- Товары
- Заказы
- Правила доступа

### Принцип работы

Для каждой роли задаются правила доступа к определённому ресурсу.

Поддерживаются следующие действия:

- просмотр своих объектов;
- просмотр всех объектов;
- создание объектов;
- изменение своих объектов;
- изменение всех объектов;
- удаление своих объектов;
- удаление всех объектов.

Проверка прав выполняется через отдельную функцию проверки доступа.

## API

### Регистрация

```http
POST /api/auth/register/
```

### Авторизация

```http
POST /api/auth/login/
```

### Выход из системы

```http
POST /api/auth/logout/
```

### Профиль пользователя

Получение профиля:

```http
GET /api/auth/profile/
```

Изменение профиля:

```http
PATCH /api/auth/profile/
```

Удаление профиля:

```http
DELETE /api/auth/profile/
```

### Товары

```http
GET /api/auth/products/
```

### Заказы

```http
GET /api/auth/orders/
```

### Правила доступа

Получение списка правил:

```http
GET /api/auth/access-rules/
```

Изменение правила:

```http
PATCH /api/auth/access-rules/<id>/
```

## Особенности реализации

В качестве механизма аутентификации используются JWT-токены.

Для хранения активных пользовательских сессий реализована отдельная таблица токенов авторизации.

Авторизация построена на основе ролевой модели доступа (RBAC). Для каждой роли задаются права на выполнение определённых действий над ресурсами системы.

Права доступа хранятся в базе данных и могут изменяться пользователем с ролью «Администратор» через API.
